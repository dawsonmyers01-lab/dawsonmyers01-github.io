Week 1: Newsletter launch, GitHub Pages, and what I learned

This week was about getting set up and actually publishing. The main lesson was that “live” is not automatic. You have to build the files correctly and make sure the homepage links everything.

Summary

Here is what we covered: getting started with GitHub, using AI as a coding partner, building the newsletter files, and deploying to GitHub Pages. Then we used a checklist to confirm everything worked.

Key takeaways

1) Repository setup matters.
GitHub Pages is picky about structure. The homepage must be named index.html and it must exist in the place your Pages settings point to (usually the repo root).

2) AI is more useful when you give it structure.
Instead of “make me a website,” it worked better to ask for specific files and required elements like a post list, correct filenames, and consistent links.

3) Publishing is two steps.
Step 1 makes the post file exist.
Step 2 puts the post on the homepage so people can actually find it.
